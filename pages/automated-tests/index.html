How does it work?

  1. Initially, the repository webcardinal-tests is cloned inside .dev/webcardinal-tests.
  2. Then by octopus runner, tests are moved in the development root of the component library, in test directory.
    (e.g .dev/webcardinal/.webcardinal/components/webcardinal-core/test).
  3. At that location jest and puppeteer will be installed (automatically by octopus).

In a nutshell, a Chromium instance is used for E2E testing.

What is WebCardinal Testing Environment?

A test environment is a WebCardinal application designed specifically for testing. It contains pages, templates, controllers, etc and it should supply auxiliary methods and helpers.

How to run current tests

From the root of this application run the following command: npm run test # npm t

Testing Environment of WebCardinal

    NOTES:
  • Automated Environment will uses the same webcardinal bundle (bundle-all) as current workspace (webcardinal-app-tester).
  • When tests are running Stencil creates a temporarily distribution (using Stencil Dev Server ⚡) .
